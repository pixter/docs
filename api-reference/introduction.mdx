Bem-vindo à documentação oficial da **API de Frontend da Sinerlog**. Esta documentação tem como objetivo orientar desenvolvedores e integradores sobre **como consumir os dados e serviços expostos ao frontend**, garantindo consistência, segurança e alinhamento com as regras de negócio da plataforma Sinerlog.

---

## 🎯  Propósito da API

A API frontend foi desenhada para:

- Expor **dados da sessão autenticada** (usuário, permissões, contas, aplicações).
- Fornecer **informações contextuais** para personalizar a experiência da interface (ex: menus, roles, configurações de exibição).
- Permitir **operações específicas** que não dependem diretamente da API de backend, mas que afetam a UI.
- Servir como ponte entre **Auth0 (autenticação)** e o renderizador do frontend (React, Next.js, etc).

---

## 🔐 Acesso e segurança

Todas as rotas são protegidas por autenticação OAuth2, usando **Auth0 com idToken JWT**.\
O token deve ser incluído no header `Authorization` como:

```
Authorization: Bearer <seu_token_aqui>
```

A API validará o token antes de responder a qualquer solicitação.

## 🚧 Aviso importante

Esta API **não deve ser confundida com a API de backend** da Sinerlog, que lida com lógica de negócio, persistência e integrações externas.\
A API frontend é voltada exclusivamente para **suporte à camada de apresentação** e segurança de interface.

---

## 🧱 Requisitos

- Cliente OAuth2 válido (via Auth0)
- Frontend compatível com JWT
- Conexão HTTPS

---

## 🛠️ Stack envolvida

- Auth0 (Custom DB \+ Actions)
- Next.js 14 / React 18
- Token JWT com claims customizados
- Integração com sessão segura
- Supabase opcional para dados adicionais
