## Visão Geral

O **Sinerlog Frontend Global** é uma aplicação web moderna desenvolvida para unificar todos os frontends da Sinerlog em uma única plataforma. Este projeto representa a evolução da arquitetura frontend da empresa, focando em padronização, componentização e dinamismo.

### Objetivo Principal

O objetivo central deste frontend é consolidar todos os sistemas da Sinerlog em uma única aplicação, eliminando a fragmentação de interfaces e proporcionando uma experiência consistente para usuários e desenvolvedores.

### Principais Benefícios

- **Unificação**: Todos os sistemas em uma única aplicação
- **Padronização**: Componentes e padrões consistentes
- **Componentização**: Reutilização máxima de código
- **Dinamismo**: Interface adaptável e responsiva
- **Manutenibilidade**: Arquitetura limpa e organizada

## Arquitetura

### Filosofia Arquitetural

O projeto adota uma arquitetura **feature-based** com clara separação de responsabilidades, inspirada nos princípios da **Clean Architecture** e do padrão **MVVM** (Model-View-ViewModel).

### Princípios Fundamentais

- **Baixo Acoplamento**: Cada funcionalidade independe ao máximo das outras
- **Alta Coesão**: Código relacionado permanece agrupado
- **Testabilidade**: Estrutura que facilita testes unitários e de integração
- **Escalabilidade**: Facilita adição de novas funcionalidades

### Organização por Features

Em vez de organizar o código somente por camadas técnicas, ele é estruturado em torno de **features** ou módulos de funcionalidades. Isso garante:

- **Isolamento**: Problemas podem ser isolados e resolvidos rapidamente
- **Desenvolvimento Paralelo**: Múltiplos desenvolvedores podem trabalhar simultaneamente
- **Manutenção Simplificada**: Alterações têm impacto mínimo em outras partes

## Desenvolvimento

### Pré-requisitos

- **Node.js**: Versão 18 ou superior
- **Bun**: Runtime JavaScript (recomendado)
- **Git**: Controle de versão

### Comandos Principais

```bash
# Instalar dependências
bun install

# Executar em desenvolvimento
bun run dev

# Executar testes
bun run test

# Build para produção
bun run build

# Formatar código
bun run format
```

### Variáveis de Ambiente

O projeto utiliza variáveis de ambiente para configurações sensíveis:

```env
# Auth0
AUTH0_SECRET=
AUTH0_BASE_URL=
AUTH0_ISSUER_BASE_URL=
AUTH0_CLIENT_ID=
AUTH0_CLIENT_SECRET=

NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
```
